# Plan: cli-extractor
_Gerado em: 2026-03-06_

## Problema

Implementar a lógica de extração do comando `agent-index generate`: ler CLAUDE.md + `.claude/rules/*.md`,
extrair hot files, invariantes e stack, e gerar `.agent-index.md` como documento hot-memory curado
para agentes — conciso, não copy-paste, com informação genuinamente não-óbvia.

## Assunções

- [assumed][background] O extrator é project-type-agnostic: lê CLAUDE.md + rules/ genéricos; tipos específicos (Swift, TS) ficam em `cli-project-types`
- [assumed][background] Hot files = file paths referenciados em CLAUDE.md e rules/ (heurística: linhas com backtick path-like patterns)
- [assumed][background] Invariantes = frases imperativas extraídas de rules/ (padrões: "Never", "Always", "nunca", "sempre", "must")
- [assumed][background] Stack = seção `## Stack` do CLAUDE.md se existir; fallback: detectar por extensões de arquivos mencionadas
- [assumed][background] Output é `.agent-index.md` na raiz do project-root por padrão (já definido no scaffold)

## Deliverables

### Deliverable 1 — Walking Skeleton

**O que faz:** Módulo `extractor.ts` que lê CLAUDE.md + rules/, retorna dados estruturados; `generate.ts` usa o extrator e escreve `.agent-index.md`.

**Critério de done:** `node dist/index.js generate --project-root .` roda na raiz do repo e gera `.agent-index.md` com seções não-vazias.

### Deliverable 2 — Testes e robustez

**O que faz:** Testes unitários para `extractor.ts` usando fixtures; casos de borda (sem rules/, CLAUDE.md ausente, diretório vazio).

**Critério de done:** `npm test` passa com cobertura de todos os casos de borda.

## Arquivos a modificar/criar

- `cli/src/extractor.ts` — criar: lógica de extração (readClaudeMd, readRules, extractHotFiles, extractInvariants, extractStack, formatOutput)
- `cli/src/commands/generate.ts` — editar: substituir stub pelo uso do extrator
- `cli/src/commands/generate.test.ts` — editar: adicionar testes de integração com fixtures
- `cli/src/extractor.test.ts` — criar: testes unitários do extrator

## Passos de execução

1. Criar `cli/src/extractor.ts` com as funções de leitura e extração [Deliverable 1]
2. Editar `cli/src/commands/generate.ts` para usar o extrator e escrever o arquivo de saída [Deliverable 1]
3. `cd cli && npm run build && node dist/index.js generate --project-root ..` — verificar output gerado [Deliverable 1]
4. Criar `cli/src/extractor.test.ts` com fixtures inline e casos de borda [Deliverable 2]
5. Editar `cli/src/commands/generate.test.ts` para adicionar testes de integração [Deliverable 2]
6. `cd cli && npm run build && npm test` — todos os testes passam [Deliverable 2]

## Formato do .agent-index.md gerado

```markdown
# Agent Index
_Generated by agent-index v0.1.0 — <timestamp>_

## Stack
- <item extraído da seção ## Stack do CLAUDE.md>

## Hot Files
- `path/to/file` — <contexto da linha onde foi mencionado>

## Invariants
- <frase imperativa extraída de rules/*.md> (fonte: <nome-do-arquivo>)
```

Regras de extração:
- Hot files: regex `` `[./\w-]+\.\w+` `` em CLAUDE.md e rules/; deduplicar; máx 15 itens
- Invariantes: linhas que contêm "Never", "Always", "nunca", "sempre", "must", "Nunca", "Sempre" em rules/; máx 20 itens
- Stack: lista abaixo de `## Stack` no CLAUDE.md; se ausente, omitir seção
