# Contributing

This is a spec, not a library. Contributions are feedback, examples, and proposed conventions — not code (for now).

## How to give feedback

**Read the spec first:** [rmolines.github.io/agent-native-dev-conventions/spec/v0.1/](https://rmolines.github.io/agent-native-dev-conventions/spec/v0.1/)

Then open an issue using one of the two templates:

- **Principle feedback** — agree, disagree, refine, or add nuance to one of the 7 conventions
- **Missing convention** — something agents struggle with that isn't covered

The most useful feedback includes your stack, project size, and a concrete example. "This doesn't work for monorepos because X" is more useful than "I don't like convention III".

## What happens to feedback

Issues are the primary input for v0.2. Recurring themes across multiple stacks → candidate for a revised or new convention. Single-context edge cases → noted in the spec as exceptions or variants.

## What's out of scope for now

- Tooling (the CLI comes in M2)
- Support for agents other than Claude Code
- Generalizing conventions before they're validated on 2+ real projects

If you're unsure whether something fits, open the issue anyway.
